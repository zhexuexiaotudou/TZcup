# 赛题要求到仿真模块的映射

## 1. 必须覆盖的赛题能力

| 赛题能力/指标 | 仿真实现 | 证据 |
|---|---|---|
| 多场景全覆盖清扫 | OpenNav Coverage + Nav2；清扫 footprint 轨迹并集 | 覆盖率、漏扫率、重复率、轨迹图 |
| 垃圾识别、自动清扫 | RGB-D + 仿真真值；检测节点；清扫状态机 | Precision/Recall、目标位置误差、演示 rosbag |
| 定时循迹清扫 | Waypoint/Route + 定时任务触发 | 任务日志、路线完成率 |
| 定点清扫 | NavigateToPose + 局部清扫动作 | 到点误差、任务完成时间 |
| 轻度积水 ≤1 cm | 低摩擦薄层区域和速度限制 | 通过率、横向误差、打滑率代理量 |
| 落叶堆积 ≤3 cm | 低矮密集目标区，后续加入阻力/检测难度 | 识别率、清扫完成率 |
| 清扫效率 ≥3500 m²/h | 由作业宽度、有效速度、重复率计算 | 有效面积/总时间 |
| 定位精度 ≤50 mm | Gazebo ground truth 对 AMCL/EKF/SLAM 轨迹 | XY RMSE、P95、最大误差 |
| 清洁宽度 ≥600 mm | URDF 清扫 footprint 0.65 m | 模型参数和可视化 |
| 建图面积 ≥20000 m² | 小场景功能验证 + 大场景 headless 压测 | 地图面积、内存和实时率 |
| 动态避障 | Nav2 costmap/controller/collision monitor | 成功率、最小障碍距离 |
| 边界防护 | keepout filter / geofence | 越界次数=0 |
| 故障应急，制动 ≤1 s | `/emergency_stop` 高优先级速度门控 | 指令到速度归零延迟 |
| 多模态交互 | 后续语音 + APP；仿真阶段先定义统一任务 JSON | 指令回放测试 |
| 大模型任务分解 | 后续自然语言 -> 结构化任务 -> BT | 任务集准确率 |
| 实时动态调整动作序列 | BT 恢复、重新规划、任务暂停/恢复 | 障碍注入实验 |
| 抓取 | 预留机械臂安装位；后续 MoveIt 2 最小演示 | 抓取成功率 |

## 2. 指标计算口径

### 2.1 清扫覆盖率

\[
R_{coverage} = \frac{A_{cleaned} \cap A_{target}}{A_{target}}
\]

其中 `A_cleaned` 由机器人清扫作业 footprint 沿轨迹扫掠后求并集。

### 2.2 漏扫率

\[
R_{miss} = 1 - R_{coverage}
\]

### 2.3 重复清扫率

\[
R_{overlap} = \frac{A_{swept,total} - A_{cleaned,union}}{A_{target}}
\]

### 2.4 有效清扫效率

\[
E = \frac{A_{cleaned,union}}{T} \times 3600
\]

单位为 `m²/h`。必须同时报告目标区大小、作业宽度、平均速度和重复率，不能只按理论速度计算。

### 2.5 定位误差

\[
e_i = \sqrt{(x_i-x_i^{gt})^2+(y_i-y_i^{gt})^2}
\]

报告 RMSE、P95 和最大值。仿真真值与估计轨迹必须时间同步。

### 2.6 避障成功率

\[
R_{avoid}=\frac{N_{无碰撞且完成任务}}{N_{总试验}}
\]

每类场景至少重复 20 次，最终总次数建议不少于 100 次。

### 2.7 急停响应

从 `/emergency_stop=true` 的时间戳，到底盘线速度和角速度均低于阈值的时间差。阈值和采样频率必须写进报告。

## 3. 不能直接由初期仿真证明的指标

- 日均故障率 <1%；
- 新手 3 天掌握；
- 易损件更换 ≤3 步；
- 实车尘箱承载强度；
- 实际积水和落叶清洁效果。

这些指标应使用长期运行测试、用户测试、机械设计和实车实验补齐，不能用短时仿真替代。
